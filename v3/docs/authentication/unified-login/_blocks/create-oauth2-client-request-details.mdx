   Creates an OAuth2 client

 <H2>Request</H2>

 <H3>Body Schema</H3>

| Name                                       | Type                      | Description                                                                                                                                                                                     | Required | Default Value |
|--------------------------------------------|--------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------|---------------|
| `clientName`                               | `string`                 | A human-readable name of the client that will be used for identification.                                                                                                                       | Yes      | -          |
| `responseTypes`                            | `array` of `ResponseType` | The types of responses your client expects from the [**Authorization Server**](/docs/authentication/unified-login/introduction#authorization-server). Use `code` to receive an Authorization Code that will be exchanged for an OAuth2 Access Token. | Yes      | -    |
| `grantTypes`                               | `array` of `GrantType`   | The grant types that the Client will use. Supports `authorization_code` for exchanging Authorization Code and `refresh_token` for exchanging OAuth2 Refresh Token.                            | Yes      | - |
| `tokenEndpointAuthMethod`                  | `string`                 | Indicates that the process of obtaining an OAuth2 Access Token will not make use of the client secret.                                                                                         | Yes      | -      |
| `redirectUri`                              | `array` of `string`      | List of URIs where the Authorization Server will redirect after authorization. Can include mobile/desktop deep links. Must be exact URLs without wildcards.                                    | Yes      |  -  |
| `scope`                                    | `string`                 | Space-separated string of scopes the Client will request. Include `offline_access` to enable OAuth2 Refresh Token usage.                                                                      | Yes      | - |
| `authorizationCodeGrantAccessTokenLifespan` | `Time Duration`          | OAuth2 Access Token lifespan when using the Authorization Code grant flow.                                                                                                                      | No       | `"1h"`        |
| `authorizationCodeGrantIdTokenLifespan`     | `Time Duration`          | OAuth2 ID Token lifespan when using the Authorization Code grant flow.                                                                                                                          | No       | `"1h"`        |
| `authorizationCodeGrantRefreshTokenLifespan`| `Time Duration`          | OAuth2 Refresh Token lifespan when using the Authorization Code grant flow.                                                                                                                     | If `refreshTokenGrantRefreshTokenLifespan` is also set        | `"30d"`       |
| `refreshTokenGrantRefreshTokenLifespan`     | `Time Duration`          | OAuth2 Refresh Token lifespan when using the Refresh Token grant flow. Must match `authorizationCodeGrantRefreshTokenLifespan`.                                                                | If `authorizationCodeGrantRefreshTokenLifespan` is also set       | `"30d"`       |
| `enableRefreshTokenRotation`               | `boolean`                | Indicates that the refresh token is a one-time use. Set it to `false` to disable refresh token rotation.                                                                         | No       | `true`        |


<H4>Time Duration</H4>

A string value that signifies time duration in milliseconds, seconds, minutes or hours: `"2000ms"`, `"60s"`, `"30m"`, `"1h"`.

<H3>Example</H3>

```bash
curl -X POST ^{coreInfo.uri}/recipe/oauth/clients \
  -H "Content-Type: application/json" \
  -H "api-key: ^{coreInfo.key}" \
  -d '{
      "clientName": "<YOUR_CLIENT_NAME>",
      "responseTypes": ["code"],
      "grantTypes": ["authorization_code", "refresh_token"],
      "tokenEndpointAuthMethod": "none",
      "scope": "offline_access <custom_scope_1> <custom_scope_2>",
      "redirectUri": ["https://<YOUR_APPLICATION_DOMAIN>/oauth/callback"],
    }'
```

<H2>Response</H2>

<H3>200</H3>

The client has been successfully created.

<H3>Response Schema</H3>

| Property    | Type                             | Description                                   |
|-------------|----------------------------------|-----------------------------------------------|
| `clientName`        | `string`                         | Unique identifier for the user.               |
| `clientId`  | `Object` | The initial data that you have added through the `/bulk-import/add-users` endpoint.|
| `callbackUrls` | `string` (`"NEW"`, `"PROCESSING"`, `"FAILED"`) | Current status of the user.                   |

<H4>Example</H4>

```json
{
    "clientName": "<YOUR_CLIENT_NAME>",
    "clientId": "<CLIENT_ID>",
    "callbackUrls": ["https://<YOUR_APPLICATION_DOMAIN>/oauth/callback"],
}
```
