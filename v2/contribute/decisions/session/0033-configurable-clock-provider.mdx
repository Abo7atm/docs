# Make the timestamp provider configurable on the FE

<DecisionHeader status="proposed" lastUpdate="2023-01-24" created="2022-12-24" deciders={["rishabhpoddar", "porcellus"]} proposedBy={["porcellus"]} />

## Context and Problem Statement

As a consequence of [this decision](./0032), we will start using a timestamp provider to get dates. We need to decide if we want to make it configurable.

## Considered Options

* Not configurable
* Configurable on the FE
* Configurable on the BE and FE

## Decision Outcome

We've decided to make it configurable on the FE only:
- We are making other built-ins configurable there (e.g.: window and cookie handlers) but they are not mirrored in the BE anyway
- We assume that the BE clock is controlled by the application devs
- It would be very rarely useful outside of testing

### Not configurable

<ArgumentPro> Does not expand interface </ArgumentPro>
<ArgumentCon> We allow other "globals" to be mocked/replaced on the FE </ArgumentCon>

### Configurable on the FE

<ArgumentPro> Allow apps to use custom implementation </ArgumentPro>
<ArgumentPro> Matches how we allow other "globals" to be mocked/replaced </ArgumentPro>
<ArgumentPro> Could help with FE testing </ArgumentPro>
<ArgumentCon> Doesn't help with BE testing </ArgumentCon>

### Configurable on the FE and BE

<ArgumentPro> Allow apps to use custom implementation </ArgumentPro>
<ArgumentPro> Matches how we allow other "globals" to be mocked/replaced on the FE </ArgumentPro>
<ArgumentPro> Should speed up testing by a lot </ArgumentPro>
<ArgumentCon> BE option would be very rarely used outside testing </ArgumentCon>
